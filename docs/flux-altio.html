Mission — Broadcast
Le proprio publie, les prestataires postulent
7 étapes ▼
Vue complète
Côté proprio
Côté presta
pending
Création de la mission
Le proprio choisit : propriété, type (ménage/linge/maintenance), date, tarif fixe, description. La mission est visible par tous les prestataires de la zone.
Propriétaire → createMission()
pending
Candidatures des prestataires
Les prestataires voient la mission et postulent avec un tarif proposé + message. Max 3 candidatures par mission. Vérification SIRET + RC Pro obligatoire.
Prestataire → applyToMission()
assigned
Acceptation d'une candidature
Le proprio voit les candidatures (nom, note, tarif). Il accepte → la mission passe en "assigned". Le prestataire est notifié par push.
Propriétaire → handleApplication(id, 'accepted')
in_progress
Démarrage de la mission
Le prestataire clique "Démarrer". Il voit l'adresse complète, le code d'accès si fourni.
Prestataire → startMission()
awaiting_payment
Fin de mission + photos
Le prestataire uploade les photos de fin (min. 1 si require_photos=true), puis clique "Terminer". La mission attend la validation du proprio.
Prestataire → completeMission()
paid
Validation + Paiement
Le proprio vérifie les photos, clique "Valider & Payer". Stripe capture le montant (tarif × 1.10 pour le proprio, × 0.90 pour le presta). Facture générée automatiquement.
Propriétaire → validateMission() → Stripe → completeMissionPayment()
paid
Notation
Le proprio est invité à noter le prestataire (1-5 étoiles + commentaire). Il peut aussi l'ajouter en favori.
Propriétaire → submitReview()
⚠️ À tout moment avant le paiement, le proprio ou le presta peut annuler → cancelMission() → statut cancelled
🔀 Variante : Aucune candidature
Si personne ne postule, le proprio peut cliquer "Élargir la zone" → expandMissionZone() → relance la recherche à 30km. Si toujours rien → la mission expire (expired) et peut être republiée.
🔀 Variante : Mode devis
Au lieu du tarif fixe, le prestataire soumet un devis détaillé avec lignes (main d'œuvre, fournitures, etc.). Le proprio accepte ou refuse. Si accepté → paiement en pré-autorisation → travaux → capture finale.

Urgence
Plomberie, électricité, serrurerie — enchères + devis sur place
10 étapes ▼
Vue complète
Côté proprio
Côté presta
bids_open
1. Création de l'urgence
Le proprio choisit propriété + type (plomberie/électricité/serrurerie/jacuzzi/réparation) + description + photos. Countdown 24h lancé. Notifs push envoyées aux prestas de la zone (favoris d'abord, 2h de priorité).
Propriétaire → createEmergency()
bids_open
2. Enchères des prestataires
Chaque presta soumet un bid : coût déplacement + coût diagnostic + heure d'arrivée estimée. Plusieurs prestas peuvent enchérir.
Prestataire → submitEmergencyBid()
bid_accepted
3. Le proprio choisit un bid
Le proprio voit les bids avec badges : "Arrive le plus vite" / "Moins cher" / "Mieux noté". Il accepte → paiement Stripe immédiat (déplacement + diagnostic).
Propriétaire → acceptEmergencyBid() + Stripe
displacement_paid
4. En route
Le presta se déplace. Le proprio voit "En route" avec l'ETA.
on_site
5. Arrivée sur place
Le presta clique "Je suis arrivé". Il diagnostique le problème.
Prestataire → markEmergencyArrived()

⚡ Deux chemins possibles :

Chemin A — Pas besoin de travaux supplémentaires
La première heure (déplacement + diagnostic) suffit. Le presta clique "Terminer" → completeEmergency() → statut completed. Le paiement du bid était la totalité. Terminé.
Chemin B — Travaux nécessaires → Devis
quote_submitted
6. Le presta crée un devis
Lignes détaillées : main d'œuvre, pièces, déplacement. TVA calculée (20% standard, 10% rénovation, 0% auto-entrepreneur). Frais Altio 10% en plus.
Prestataire → submitQuoteWithLines()
quote_accepted
7. Le proprio accepte le devis
Il voit le détail HT/TVA/TTC + frais. Il accepte → Stripe en PRÉ-AUTORISATION (pas capturé immédiatement).
Propriétaire → acceptEmergencyQuote() + Stripe hold
in_progress
8. Travaux en cours
Le presta fait les travaux. Le paiement est bloqué sur la carte du proprio.
completed
9. Fin → Capture du paiement
Le presta clique "Terminer" → le paiement pré-autorisé est capturé → completed.
Prestataire → completeEmergencyWithCapture()
completed
10. Notation
Le proprio note le presta. Fin du flux.
🔀 Si le proprio refuse le devis
Statut → quote_refused. Le presta peut resoumettre un nouveau devis. Le cycle reprend à l'étape 6.

Devis
Création ligne par ligne, acceptation, paiement
5 étapes ▼
quote_submitted
Création du devis
Le presta accède à /quote/create. Il ajoute des lignes : type (main d'œuvre / fournitures / déplacement / diagnostic / custom), description, quantité, unité (heure/m²/forfait), prix unitaire HT. TVA calculée auto (20% standard, 10% rénovation, 0% micro-entrepreneur).
quote_sent
Envoi au propriétaire
Le devis est envoyé. Le proprio reçoit une notification.
quote_accepted
Acceptation + pré-autorisation
Le proprio voit : sous-total HT, TVA, total TTC, frais Altio 10%, grand total. Il coche la case d'acceptation. Stripe bloque le montant (capture manuelle).
assigned / in_progress
Travaux
Le presta effectue les travaux. Le paiement est bloqué.
completed / paid
Complétion + capture
Le presta termine → le paiement pré-autorisé est capturé. Facture générée.
🔀 Si le proprio refuse le devis
Statut → quote_refused. Le presta peut resoumettre un nouveau devis ou la mission/urgence peut être annulée.

Paiements Stripe
4 points de paiement dans l'app — commissions 10%+10%
4 flux ▼
💳 1. Mission — Valider & Payer
Quand : statut awaiting_payment
Mode : capture immédiate
Montant proprio : tarif × 1.10 (inclut 10% frais Altio)
Montant presta : tarif × 0.90 (après 10% commission)
→ createPaymentIntent() → Stripe Sheet → completeMissionPayment()
💳 2. Urgence — Accepter un bid
Quand : proprio accepte un bid en bids_open
Mode : capture immédiate
Montant : déplacement + diagnostic + frais
→ acceptEmergencyBid()
💳 3. Urgence — Accepter un devis
Quand : proprio accepte devis en quote_sent
Mode : PRÉ-AUTORISATION (capture manuelle plus tard)
Montant : total TTC + frais Altio
→ acceptEmergencyQuote()
💳 4. Urgence — Capture finale
Quand : presta termine les travaux (quote_accepted)
Mode : capture du montant pré-autorisé
→ completeEmergencyWithCapture()

Autres flux
Chat, notifications, avis, favoris, planning, iCal
6 flux ▼
💬 Chat
Messagerie temps réel entre proprio et presta assigné. Supabase Realtime. Accessible depuis le détail mission/urgence.
🔔 Notifications
Double système : push via Edge Function send-push + in-app stockées en DB. Triggers automatiques sur chaque changement de statut. Invalidation cache via useRealtimeSync.
⭐ Avis / Reviews
Après chaque mission payée ou urgence terminée. 1-5 étoiles + commentaire. Affiché sur le profil presta. Auto-popup après paiement.
❤️ Favoris
Le proprio ajoute un presta en favori. Les favoris ont 2h de priorité sur les broadcasts et urgences. Table favorite_providers.
📅 Planning prestataire
Vue calendrier des missions du jour avec statuts colorés. Écran (provider)/planning.tsx + hook useProviderSchedule().
📆 Sync iCal
Edge Function sync-ical + cron. Synchro des réservations Airbnb/Booking via URL .ics. Actuellement : crée des missions auto (en cours de redesign pour juste notifier le proprio).
